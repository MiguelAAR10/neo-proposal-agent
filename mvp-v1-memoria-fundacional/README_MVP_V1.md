# MVP V1 - Memoria Fundacional

Este directorio reúne la base documental histórica de la Version 1 (V1).

Objetivo:
- Preservar aprendizajes reales del desarrollo V1.
- Dar contexto operativo a los agentes que reconstruyen V2.
- Evitar repetir errores técnicos y de organización.

## Documentos incluidos

1. `PROJECT_DETAILS.md`
- Arquitectura y lógica funcional consolidada de V1.

2. `BITACORA_APRENDIZAJES.md`
- Lecciones aprendidas, problemas reales y cómo se resolvieron.

3. `RADIOGRAFIA_PROYECTO.md`
- Diagnóstico del caos previo y prioridades de ordenamiento.

## Uso recomendado para agentes de desarrollo

1. Leer primero `BITACORA_APRENDIZAJES.md` para conocer riesgos históricos.
2. Leer `PROJECT_DETAILS.md` para entender el baseline funcional.
3. Usar `RADIOGRAFIA_PROYECTO.md` como checklist de anti-patrones a evitar.

## Relación con V2

- La especificación oficial activa para reconstruir V2 está en `neo-proposal-specs/`.
- Este directorio es contexto fundacional V1, no contrato funcional de V2.
